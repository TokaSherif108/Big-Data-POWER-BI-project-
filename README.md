# PowerBI-Project
Through this project, I was introduced to technologies and concepts that completely shifted my perspective, including:

• Working with Databricks  
• Handling large datasets from 7 CSV files and 2 JSON files with different formats  
• Understanding the importance of performance optimization, not just dashboard design  
• Learning why business owners cannot wait for slow dashboards to load  
• Using DirectQuery instead of relying only on imported data  
• Working with Delta Tables and optimizing storage schemas to improve performance  
• Creating measures strategically in Databricks before connecting to Power BI  
• Managing cloud-based data workflows instead of downloading datasets locally  

The project itself was based on restaurant chain data, and this was one of the first times I truly tried to think from a business perspective instead of only focusing on charts and visuals.

I started asking questions like:

• Which branch generates the highest revenue?  
• Which branch might have operational problems or losses?  
• Are discounts affecting profitability?  
• Which products perform the best and worst?  
• Are customers returning or not?  

I also learned how important data cleaning and data quality are before starting any analysis.

For example:

• I found duplicated order IDs and had to think about how to handle them properly  
• I created a new primary key using a Series ID  
• I found negative price values and handled them through analysis and averaging techniques  
• I tried to understand whether the issue came from data entry problems or from the business process itself
